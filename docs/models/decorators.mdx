This page documents every decorator that is available in Sequelize.

<DecoratorInfo />

## `@Table`

The `@Table` decorator is used to configure a model. 
It is used on a model class, and takes an object as parameter.

See [`ModelOptions`](pathname:///api/v7/interfaces/ModelOptions.html) for a list of all available options.

```ts
@Table({
  tableName: 'users',
  timestamps: false,
})
class User extends Model<InferAttributes<User>, InferCreationAttributes<User>> {}
```

Using this decorator is completely optional, you only need to use it if you want to configure one of the options of your model.

## Attribute Decorators

### `@Attribute`

The `@Attribute` decorator is used to add an attribute to a model.
It is used on an instance property, and can be used in multiple ways.

By passing the type of the attribute as parameter:

```ts
class User extends Model<InferAttributes<User>, InferCreationAttributes<User>> {
  @Attribute(DataTypes.STRING)
  declare firstName: string | null;
}
```

Or by passing a configuration object as parameter, which allows you to configure all available attribute definition options.  
See [`ModelAttributeColumnOptions`](pathname:///api/v7/interfaces/ModelAttributeColumnOptions.html) for a list of all available options.

```ts
class User extends Model<InferAttributes<User>, InferCreationAttributes<User>> {
  @Attribute({
    type: DataTypes.STRING,
    allowNull: false,
  })
  declare firstName: string;
}
```

### `@AllowNull`

:::info

Attributes are nullable by default.  
This decorator has been added in previsions of the implementation of [this feature request](https://github.com/sequelize/sequelize/issues/14260), 
which will make attributes non-null by default.

:::

The `@AllowNull` decorator is used make an attribute nullable, 
it is a shortcut for setting the [`allowNull`](pathname:///api/v7/interfaces/ModelAttributeColumnOptions.html#allowNull) option of the `@Attribute` decorator to true.

```ts
class User extends Model<InferAttributes<User>, InferCreationAttributes<User>> {
  @Attribute(DataTypes.STRING)
  @AllowNull
  declare firstName: string | null;
}
```

### `@NotNull`

The `@NotNull` decorator is used make an attribute not nullable,
it is a shortcut for setting the [`allowNull`](pathname:///api/v7/interfaces/ModelAttributeColumnOptions.html#allowNull) option of the `@Attribute` decorator to false.

```ts
class User extends Model<InferAttributes<User>, InferCreationAttributes<User>> {
  @Attribute(DataTypes.STRING)
  @NotNull
  declare firstName: string;
}
```

### `@PrimaryKey`

The `@PrimaryKey` decorator is used to make an attribute a primary key,
it is a shortcut for setting the [`primaryKey`](pathname:///api/v7/interfaces/ModelAttributeColumnOptions.html#primaryKey) option of the `@Attribute` decorator to true.

```ts
class User extends Model<InferAttributes<User>, InferCreationAttributes<User>> {
  @Attribute(DataTypes.INTEGER)
  @PrimaryKey
  declare id: number;
}
```

:::info

Sequelize automatically adds an auto-incremented integer attribute called `id` as primary key if none is specified.  
Learn more about this in the [Advanced model definitions page](./advanced.mdx#prevent-creating-a-default-pk-attribute).

:::

### `@AutoIncrement`

The `@AutoIncrement` decorator is used to make an attribute auto-increment,
it is a shortcut for setting the [`autoIncrement`](pathname:///api/v7/interfaces/ModelAttributeColumnOptions.html#autoIncrement) option of the `@Attribute` decorator to true.

Some dialects require the field to be a primary key.

```ts
class User extends Model<InferAttributes<User>, InferCreationAttributes<User>> {
  @Attribute(DataTypes.INTEGER)
  @PrimaryKey
  @AutoIncrement
  declare id: number;
}
```

### `@Default`

The `@Default` decorator is used to set a default value for an attribute,
it is a shortcut for setting the [`defaultValue`](pathname:///api/v7/interfaces/ModelAttributeColumnOptions.html#defaultValue) option of the `@Attribute` decorator.

```ts
class User extends Model<InferAttributes<User>, InferCreationAttributes<User>> {
  @Attribute(DataTypes.STRING)
  @Default('John Doe')
  declare firstName: string;
}
```

### `@ColumnName`

The `@ColumnName` decorator is used to set the name of the column in the database,
it is a shortcut for setting the [`columnName`](pathname:///api/v7/interfaces/ModelAttributeColumnOptions.html#columnName) option of the `@Attribute` decorator,
which is used to set the name of the column in the database.

:::info

With a good [naming strategy](./naming-strategies.mdx) configured, you rarely need to use this decorator.

:::

```ts
class User extends Model<InferAttributes<User>, InferCreationAttributes<User>> {
  @Attribute(DataTypes.STRING)
  @ColumnName('first_name')
  declare firstName: string;
}
```

### `@Unique`

The `@Unique` decorator is used to make an attribute unique,
it is a shortcut for setting the [`unique`](pathname:///api/v7/interfaces/ModelAttributeColumnOptions.html#unique) option of the `@Attribute` decorator.

```ts
class User extends Model<InferAttributes<User>, InferCreationAttributes<User>> {
  @Attribute(DataTypes.STRING)
  // makes "firstName" unique
  @Unique
  declare firstName: string;
}
```

Head to [Validations & Constraints](./validations-and-constraints.md#unique-constraint) to learn more about unique constraints.

### `@Comment`

The `@Comment` decorator is used to set the comment on a column,
it is a shortcut for setting the [`comment`](pathname:///api/v7/interfaces/ModelAttributeColumnOptions.html#comment) option of the `@Attribute` decorator.

This is only useful if you use [`sync`](./model-synchronization.md) to create your tables.

## Validation decorators

All validation decorators are shortcuts for setting the [`validate`](pathname:///api/v7/interfaces/ModelAttributeColumnOptions.html#validate)
option of the `@Attribute` decorator. They are described in the [Validations & Constraints](./validations-and-constraints.md) page.

## Model Hook decorators

All hook decorators are described in the [Hooks](../other-topics/hooks.mdx) page.
